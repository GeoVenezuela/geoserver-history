Manages all of the administrative and configuration issues for the server.

This includes internal versioning information, server-wide configuration information, and internal tracking of the served feature types.  Wherever possible, configuration information is abstracted into JavaBeans, which reside at the top level of the config directory.

All persistent configuration information is stored in XML files local to the server installation.  These XML files are read by the groups of classes in config subdirectories, with use Castor to marshal and unmarshal the XML files into Java objects.  These Java objects are then wrapped in the JavaBeans, which abstract the configuration information into useful categories for the server.
